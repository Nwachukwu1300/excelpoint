# PostgreSQL Setup for ExcelPoint Development Environment

## Project Overview
Set up PostgreSQL database for the ExcelPoint learning enhancement platform development environment, replacing the current SQLite database.

## Requirements

### Database Setup
- Install PostgreSQL on macOS using Homebrew
- Create database named 'excelpoint_dev'
- Create user 'mmesoma' with password 'password'
- Grant necessary privileges to the user

### Python Environment
- Ensure PostgreSQL Python driver (psycopg2-binary) is installed in virtual environment
- Update requirements.txt if needed
- Test database connection

### Django Configuration
- Update Django settings to use PostgreSQL instead of SQLite
- Configure database connection using environment variables
- Test Django database connection

### Data Migration
- Start with fresh database (no data migration needed)
- Run Django migrations to create schema
- Verify all models are properly created

## Technical Specifications

### Database Configuration
- Database Name: excelpoint_dev
- Username: mmesoma
- Password: password
- Host: localhost
- Port: 5432

### Environment Variables
- DB_NAME=excelpoint_dev
- DB_USER=mmesoma
- DB_PASSWORD=password
- DB_HOST=localhost
- DB_PORT=5432

### Dependencies
- psycopg2-binary (already in requirements.txt)
- python-dotenv (already in requirements.txt)

## Success Criteria
- PostgreSQL server running locally
- Database 'excelpoint_dev' created and accessible
- Django can connect to PostgreSQL database
- All migrations run successfully
- Application starts without database errors
